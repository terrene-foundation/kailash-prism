# Stitch's DESIGN.md Protocol Bridges Design Generation and COC Code Generation

Non-obvious connection: Google Stitch's DESIGN.md protocol solves the design-code separation problem that causes "AI slop" in current codegen. The protocol is a machine-readable design spec that can be:
1. Generated by Stitch (design AI)
2. Generated by Claude (from a brief)
3. Written by a human designer
4. Extracted from an existing URL

All four paths converge at the same artifact. This artifact then drives token generation, component theming, and composition decisions. The key insight: adopt the PROTOCOL, not the tool. The protocol is the bridge between design intent and code generation. Stitch is an optional accelerator.

Stitch's MCP server (`@_davideast/stitch-mcp`) with `extract_design_context` and `generate_screen_from_text` tools could integrate into the COC workflow, but no COC command or skill currently references it. This is a concrete integration opportunity for Phase 3.
