# Four-Layer Frontend Architecture: The Engine is Agent Orchestration

The Kailash Specs-Primitives-Engines-Interface pattern maps to frontend codegen, but the engine layer is NOT a custom runtime (React/Flutter already exist). The engine is the **COC agent orchestration system** — skills, agents, commands, and YAML composition grammar that orchestrate primitive composition into screens.

This is the only defensible layer in the initiative. Everything else (tokens, components, templates) can be replicated with a Shadcn fork and a CLAUDE.md. The orchestration intelligence — knowing HOW to compose primitives according to specs — is unique institutional knowledge.

Initial analysis dismissed this as "composition guidance" (prose). The red team corrected: it must be structured, machine-readable orchestration (YAML grammar), not prose instructions. This is the highest-value artifact to build.
